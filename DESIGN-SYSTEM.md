# openPBL 设计系统

openPBL 采用“现代教育出版物 + 安静的数字课堂”语言。界面通过教学关系、阶段状态和过程证据呈现 AI，不使用机器人、发光、蓝紫渐变或装饰性 Sparkles 制造 AI 感。

## 基础 token

| 语义 | Token | 色值 |
| --- | --- | --- |
| 页面背景 | `--pbl-bg` | `#F4F2ED` |
| 内容表面 | `--pbl-surface` | `#FCFBF8` |
| 正文 | `--pbl-text` | `#1F2933` |
| 次级文本 | `--pbl-text-muted` | `#5F6B76` |
| 边框 | `--pbl-border` | `#D8D6D0` |
| 教师 | `--pbl-teacher` | `#344A6A` |
| 学生 | `--pbl-student` | `#2F6F65` |
| AI | `--pbl-ai` | `#566F7F` |
| 警告 | `--pbl-warning` | `#8A6422` |
| 危险 | `--pbl-danger` | `#8A3E3E` |

正文使用 Noto Sans SC，课程标题和数字教案章节标题使用 Noto Serif SC。字重只使用 400、500、600、700。圆角只使用 6、10、14px；普通内容不使用阴影，阴影保留给固定操作栏、Drawer、Dialog 和 Toast。

## 组件原则

- 一个页面只保留一个高强调主操作。
- 普通内容用留白、段落和细边框建立层次，不嵌套无意义卡片。
- 教师、学生、AI 的角色色只表示责任与状态，不作大面积背景。
- 所有异步操作显示保存中、已保存、失败或可重试状态。
- 危险操作使用 AlertDialog；字段编辑使用 Dialog 或 Drawer；短暂结果使用 Toast。
- 所有图标按钮提供可访问名称，触控目标不小于 44px，焦点轮廓不可被移除。

## 布局

全局无传统侧边栏。顶部为课堂上下文栏：课程身份、当前阶段/任务、保存和课堂工具。长流程底部使用统一操作栏。375px 单列，768px 可使用主内容加折叠辅助区，1280px 以上才允许项目工作空间三栏。
