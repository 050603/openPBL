# PBL 项目启发问题池设计

## 目标

学生在项目启动阶段选择的是教师设计的开放式 PBL 问题，而不是课程知识点。教师可以在备课时设置一个或多个问题，也可以在项目启动课堂中增补问题，并实时查看各问题的学生选择分布。

## 数据模型

在 `PblCourseConfig` 中增加 `inquiryQuestions: string[]`。第一题同步到课程原有的 `drivingQuestion` 字段，继续服务于课程生成、AI 提示词和既有展示管线；完整问题池用于学生选题和教师统计。采用字符串数组而不是新增数据库实体，是因为 PBL 配置已经作为结构化 JSON 持久化，且课堂阶段只要求增补、不要求重命名已有问题。

旧课程没有问题池时，以原有 `drivingQuestion` 作为唯一问题展示，不执行数据迁移。

## 交互与数据流

- 备课：教师通过可增删的问题列表设置问题池，AI 建议可以逐条加入；保存时去重并把第一题设为主驱动问题。
- 备课预览：教师视图和学生视图都展示完整问题池。
- 学生项目启动：
  - 一题：加入课堂时自动写入个人项目；既有学生进入页面时也会自动补齐。
  - 多题：学生只能从教师问题池中选择，确认后写入个人项目并完成对应准备项。
  - 无题：显示等待教师发布，不允许以知识点或自由文本代替。
- 教师项目启动：可新增问题；新增内容通过现有课程更新与实时同步管线下发。页面按问题展示选择人数和学生名单，并单列尚未选择的学生。

## 一致性

学生选题继续写入个人项目的 `topic` 字段，后续方案构思、伴学上下文和教师审阅无需改管线。教师课堂中增补问题不会覆盖已有学生选择；当题库从一题扩展为多题时，学生可以重新选择。
