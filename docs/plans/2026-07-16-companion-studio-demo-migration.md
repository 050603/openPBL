# 伴学工作室 Demo 迁移方案

## 目标

将 `C:\code\marvis-office-master` 中已经完成的 Pixi 学习空间、六名伙伴、工位动画、资料角、项目白板和过程档案迁入 OpenPBL，完全替换上一版静态课堂伴学展示层；传统任务视图继续保留。

## 架构边界

- `CompanionRuntimeProvider` 仍是唯一对话运行时，继续负责 Director、SSE、多角色回应、TTS、历史恢复、主动介入和停止生成。
- Pixi 场景只消费伙伴状态并表达动作，不保存聊天、任务或正式产物。
- `CompanionStudioWorkspace` 将 OpenPBL 的六种伙伴 ID 映射到 Demo 的六个视觉角色，使用真实任务、当前发言者、流式文本、待确认操作和完成轮次生成视觉状态。
- 任务视图与伴学现场继续位于同一个 Provider 下，通过隐藏展示层切换，切换时不销毁运行时。

## 交互映射

- 点击人物：在右侧纯白检查器中查看职责、真实状态和最近任务，并可定向分配工作。
- 点击资料角：打开知识咨询浮窗，请知知通过现有大模型接口解释概念和提供查证线索。
- 点击项目白板：打开阶段目标、最近产物和待确认操作浮窗，可进入传统任务视图继续正式编辑。
- 点击过程档案：打开真实伙伴任务、过程记录、对话和阶段产物总览。
- 顶部历史入口：显示当前阶段完整对话；底部输入框可向全组或当前伙伴发起真实请求。

## 状态真实性

视觉状态仅从以下信息派生：`runtime.phase`、`runtime.currentSpeaker`、`runtime.streamingText`、`lastCompletedRound`、`companionTasks`、`companionConfirmations` 和运行时错误。当前阶段未启用的伙伴显示为“旁听中”，不会伪装为正在执行任务，也不能接受定向请求。
