# OpenPBL 原创伴学角色替换方案

## 目标

用完全原创、非马形、非像素的课堂伴学角色替换现有马形 Agent，同时保持当前 PixiJS 场景、角色状态、围巾换色、视觉锚点、寻路、TTS 发言和动作编排不变。

## 方向比较

1. **伴学小灵（推荐）**：圆润的学习精灵，书页形耳朵代表知识，嫩芽代表项目成长，蓝色围巾作为可换色身份层。角色与 OpenPBL 课堂语义直接相关，短手脚能清晰表达翻书、写字、指白板、打字、讨论和行走。
2. **纸页机器人**：模块化身体最容易做动作，但容易显得工具化，真实小组成员感较弱。
3. **学生型人物伙伴**：协作感最直接，但六套人物身份、服装和动作的一致性成本高，也更容易与纯白柔和课堂产生视觉噪声。

选择伴学小灵时，六位伙伴继续共用一套身体动作，通过围巾颜色、名称、工位和职责形成身份差异。

## 动作与运行时兼容

新角色使用统一的 `192 × 208` RGBA 帧画布、稳定脚底基线和 `bottomCenter` 视觉锚点。动作素材按照语义生成，再映射到当前动作名称，不修改业务状态来源：

- `standby`、`selected`、`working`、`thinking`、`waiting_user`；
- 坐姿和站姿发言、倾听、赞同、追问；
- 水平行走、向上行走、离椅和坐下；
- 翻书、查资料、白板规划、头脑风暴、写记录、检查成果、展示和整理档案；
- 完成、庆祝和错误反馈。

屏幕动画和 `fc_ticket` 等场景效果继续保留为独立图层。围巾仍使用单一母版色生成，由当前运行时按伙伴配置换色，因此无需复制六套完整图集。

## 安全替换顺序

1. 确认标准人物视图，锁定轮廓、材质、身体比例、书页耳朵、嫩芽和围巾结构。
2. 生成 canonical base，先制作 `standby`、`walking_horizontal`、`talking_on_stand_a`、`reading_book` 四个代表动作。
3. 在隔离的预览资源目录接入角色开关，对课堂场景做尺寸、锚点、遮挡、工位和移动速度验证。
4. 代表动作通过后，再逐组完成全部动作；每组检查角色一致性、首尾循环、透明边缘和小尺寸可读性。
5. 最终切换默认资源清单，保留旧资源开关直到全部页面回归通过，再删除马形素材和来源依赖。

## 验收条件

- 轮廓与马、斑马、独角兽、驴等马科动物无关；
- 不使用像素、体素或 8-bit 风格；
- 角色在当前白色课堂中清晰但不抢夺学习内容注意力；
- 所有核心状态都有可识别动作，行走不存在站立漂移；
- 动作切换时脚底锚点、尺寸和围巾位置稳定；
- 六位伙伴围巾色替换后仍保持同一原创角色身份；
- 替换前后的对话、TTS、任务状态、寻路和点击交互行为一致。
