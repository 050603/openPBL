# 离线自适应课程资源池设计

## 目标

自适应课堂的运行阶段不得启动课程生成任务。教师在备课期完成主课程、补基础分支和拓展分支的全部生成与确认；学生进入课堂后，系统只根据学习证据选择已有资源并插入主课程。

非功能目标：

- 触发到资源打开不依赖模型生成延迟。
- 同一课程、同一学习证据得到可复现的路径。
- 未生成或生成失败的分支不能随课程发布。
- 教师能够预览每个成品资源并核查触发条件与学生实际值。
- 课堂资源重新生成后，触发锚点仍然稳定。

## 三种方案比较

1. 运行时实时生成：内容最灵活，但等待时间和失败概率不可接受。
2. 备课预生成、运行时失败后再生成：正常路径较快，但失败时仍会破坏课堂连续性。
3. 完整离线资源池：备课成本更高、资源组合有限，但课堂性能、可控性和可审计性最好。

采用方案 3。教师端承担生成完整性，学生端保持确定性。

## 数据与标签

每个分支资源使用结构化字段作为可执行标签：

- 教学用途：补基础 / 拓展。
- 适用层次：基础生 / 平均生 / 优秀生。
- 知识锚点：关联知识点 ID。
- 插入位置：稳定大纲节点 `afterSceneId` 与下一节点 `beforeSceneId`。
- 证据规则：层次、节点小测阈值。
- 时间规则：资源时长和最少剩余预算。
- 媒体形态：PPT 或互动资源，均包含讲稿和 TTS。
- 成品状态：未生成、生成中、已就绪、失败。

这些标签同时服务于教师界面、运行时选择和触发审计，避免维护另一套不可执行的展示标签。

## 稳定锚点

生成大纲节点 ID 是课程设计身份，最终播放器场景 ID 是资源执行身份。二者不能混用。

课堂生成器已在最终场景上写入 `outlineId`。进度接口将完成的运行时场景映射为 `completedOutlineIds`；运行时触发只使用 `outlineId`，同时保留运行时场景 ID 供诊断。教师审计显示：

- 触发要求的教学节点。
- 已完成的稳定教学节点。
- 对应播放器场景 ID。

重新生成课堂资源只会改变播放器场景 ID，不会破坏触发配置。

## 运行流程

1. 教师确认主课程页面。
2. 系统生成分支大纲，教师调整内容指导、标签和触发条件。
3. 主课堂生成后，并发生成所有已确认分支成品。
4. 教师逐项预览；任一启用分支未就绪时阻止发布。
5. 学生完成前测并获得层次。
6. 每完成一个主课节点，系统使用稳定节点 ID、层次、小测和预算评估全部候选分支。
7. 满足条件且未学习过的成品资源插入播放；完成后回到主课。
8. 不满足条件的资源不进入该学生路径，审计保留具体未满足项。

## 失败处理

- 分支生成失败：保留失败原因，教师重新生成，课程不能发布。
- 课堂资源意外缺失：学生继续主课，不现场生成；教师审计显示“分支成品资源未满足”。
- 旧进度没有稳定节点：下一次进度上报根据课堂场景自动回填全部已完成节点。
- 同一分支重复命中：已有运行记录阻止重复插入。

## 验证

- 单元测试覆盖稳定节点解析、资源缺失阻止触发、分数和时间判断。
- API 测试覆盖运行时场景到大纲节点的持久化映射。
- 组件测试覆盖教师资源状态、标签、触发审计。
- 生产构建验证客户端与服务端边界。
