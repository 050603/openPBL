# 课程模块时间模型重构设计

## 目标

修正课程核查页中课程模块生成、时间预测和时间分配的状态边界，使未生成课程模块时页面保持空状态；生成后，课程模块编辑、时间分配、课程大纲资源目标时长和最终课堂生成使用同一套时间模型。

## 现状问题

- `normalizePblTeachingOutline([])` 会补齐六个默认模块，导致未点击生成时出现伪数据。
- 页面上方多元素估算优先使用 `sceneOutlines`，课程模块卡片编辑使用 `teachingOutline`，两者更新路径不一致。
- `PblTimingFeedbackPanel` 使用 `pbl-time-estimation.ts`，`PblTimeAllocationPanel` 使用 `pbl-time-model.ts`，页面同时展示两套预测/分配结果。
- 重新生成课程模块后，已有课程大纲资源可能继续保留，造成父模块、知识点和目标时长不一致。

## 设计决策

1. `teachingOutline.length === 0` 表示课程模块尚未生成。归一化和项目主线构建对空数组返回空结果，不制造默认模块。
2. 课程核查页以当前 `teachingOutline` 为实时时间模型输入；已有二级课程大纲只作为模块内容复杂度、互动和测验信息的补充，不再替代模块输入。
3. 合并为一个“课程时间预测与分配”面板：同一份 `PblCourseTimingSnapshot` 同时展示 AI 讲授、互动、测验、教师组织、过渡、模块目标、模型估算和推荐分配。
4. 时间推荐由 `pbl-time-estimation.ts` 按六阶段模型统一计算，并提供可用于编辑的整数分钟分配结果。
5. 重新生成课程模块或知识结构导致已有二级资源失效时，清空旧课程大纲和旧时间校准快照，要求重新生成/核查。
6. 最终课程生成继续读取确认后的二级资源和模块目标时长，因此教师在核查页的调整会传递到 TTS 内容预算、资源生成和课堂时间控制。

## 验证标准

- 新课程进入课程核查页时，课程模块、项目主线、模块时间预测均为空，不显示六个默认模块。
- 点击生成课程模块后，模块和统一时间面板出现；编辑任一模块标题、教学目标、职责或时长后，预测值和组成明细立即变化。
- 应用推荐分配后，模块分钟合计等于课程总时长，项目实践按模型保持主要时间段。
- 修改模块时，已有二级资源目标时长同步缩放；重新生成模块或知识图谱时，旧二级资源不会被错误复用。
- 课程生成接口收到的 `sceneOutlines`、模块目录和 `timingPlan` 与教师最终确认结果一致。
