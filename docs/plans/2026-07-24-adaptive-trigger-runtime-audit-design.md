# 自适应分支触发修复与教师审计设计

## 已确认故障

备课阶段保存的触发锚点来自场景大纲，例如 `outline-ai-1`；课程生成后的播放器场景拥有新的 ID，例如 `scene_R2VQnAjG8H`。旧运行时只要分支配置了 `afterSceneId`，就只进行严格 ID 比较，不再使用知识点锚点，因此真实播放页永远无法命中触发点。

此外，旧判定只读取“当前完成页面”的小测分数。小测在锚点之前、互动型场景承载测验，或者锚点没有节点小测时，分数都会为空。分支自身设置的 `scoreThreshold` 也未被采用。

## 修复策略

- 首选场景 ID 精确匹配；生成前后 ID 不一致时，使用分支与播放场景共享的知识点 ID 进行稳定映射。
- 分数证据按优先级选择：当前节点小测、最近相关节点小测、课前测回退。
- 使用分支自己的分数阈值；未设置时才采用课程默认阈值。
- 互动型场景只要存在已提交的测验结果，也能计算节点得分，不再要求场景类型必须为 `quiz`。
- 时间条件同时检查分支目标时长与教师设置的最少剩余时间。

## 触发审计

每次到达相关触发点时，运行时保存一次结构化判定记录，包括：

- 触发点通过场景 ID 还是知识点映射命中；
- 分支与个体路径是否启用；
- 分支是否已经运行；
- 学生层次是否匹配；
- 分数、阈值和分数来源；
- 剩余时间和最低预算；
- 最终触发或未触发结果及原因。

记录随学生自适应学习状态持久化，最多保留最近 100 条。

## 教师界面

教师点击学生的“响应状态”即可打开触发审计。面板列出课程中的每个补基础/拓展分支：已经评估的节点逐条件显示通过或失败；尚未到达的节点展示预设条件；生成中、学习中、完成或失败的分支显示运行状态。对旧数据，如果已有播放进度但没有审计记录，界面明确提示可能是旧运行时或场景 ID 不一致导致。
