# 学生分层、分支场景与即时微课设计

## 目标

在保留现有 AI 授知的 PPT、TTS、互动、测验和多智能体体验的基础上，为新课程增加三项原生能力：

1. 开课前以 3–5 道前序知识题形成基础、标准、进阶三档学习画像。
2. 主课程知识锚点后的节点测验完成时，综合前测画像、节点得分和剩余分支预算，无感插入补基础或拓展场景。
3. proposal / make 阶段识别学生“我想学某知识”的主动请求，由伴学导演判断仅告知或系统讲解；系统讲解生成 1–2 页、2–3 分钟微课。

## 架构

课程内容中的 `adaptiveLearningPlan` 是教师确认的备课契约。它只保存前测、阈值、时间预算和候选分支大纲，不提前生成分支 PPT。学生级运行状态保存在现有 `aiLearningProgress` JSON 中，记录前测分层、节点证据、分支运行和即时微课，因此不增加独立数据库表。

具体分支和微课通过现有 `/api/openmaic/generate` 生成独立课堂快照，继续使用同一 SceneOutline、场景生成、动作、TTS 和 Stage 播放管线。独立课堂播放器以 standalone 模式运行，不写入主课程进度。进入分支前卸载主播放器，分支结束后重新挂载并从权威进度恢复主课程，避免两个播放器争用全局 Stage store。

## 备课体验

课程大纲步骤顶部新增“自适应学习路径”编辑区。教师可：

- 让 AI 根据已确认知识图谱和主课程大纲生成前测与分支大纲；
- 调整分层阈值、分支总预算、题目、正确答案、分支标题和目标；
- 查看每个分支的知识锚点、类型和预计时长；
- 显式确认整套路径。未确认的计划不会在课堂中生效。

AI 不可用时，接口返回基于知识点生成的可编辑基础方案，避免阻塞备课。

## 运行时决策

节点分支决策是确定性纯函数：

- 计划必须已启用且由教师确认；
- 必须已有前测画像；
- 当前测验场景必须关联分支锚点；
- 同一分支不得重复执行；
- 剩余预算至少 90 秒，且覆盖分支目标时长；
- 基础画像或低节点得分选择补基础；进阶画像且高节点得分选择拓展；其余继续主课程。

节点得分、分支原因和生成结果均持久化。生成失败时显示可关闭的错误状态，主课程仍可恢复。

## proposal / make 即时微课

伴学运行时只对明确的知识学习请求调用导演判断。事实、定义和短操作继续由原伴学对话回答；涉及原理、多个步骤、概念关系或项目质量的请求生成微课。微课浮层使用完整 Stage 播放器，结束后返回原项目工作区；生成与完成记录进入学生自适应状态。

## 验证

- 纯函数覆盖前测计分、三档分层、预算约束、分支选择和学习请求识别。
- 播放器回归覆盖主课堂进度与 standalone 模式的隔离。
- TypeScript 全量检查确保新数据契约、API 和组件边界一致。
- UI 重点检查备课编辑、前测、分支生成/播放/返回，以及 proposal / make 微课浮层。
