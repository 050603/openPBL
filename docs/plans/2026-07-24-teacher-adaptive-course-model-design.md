# 教师端自适应课程建模设计

## 目标

教师端不再把自适应教学理解为一组独立阈值，而是能够看到并编辑学生实际经历的完整课程模型：课前前测、固定主课程页面、页间触发器、补基础资源、拓展资源以及返回主课程的位置。授课阶段则按学生展示学习类别、类别来源和系统响应状态，并允许教师关闭个体路径或人工调整层次。

## 备课顺序

课程基本信息必须先确定，因为它定义学段、目标、课时和驱动问题。知识图谱与六模块计划随后确定课程内容边界和时间边界。主课程页面必须先于自适应路径生成，因为每个分支都要绑定真实的 `afterSceneId` 和 `beforeSceneId`。因此顺序为：

1. 基础信息
2. 知识图谱
3. 六模块与时间分配
4. 主课程页面
5. 自适应前测、触发点和分支资源
6. 评价方案

备课页面默认展开课程路径详情。自适应方案生成请求直接携带教师当前尚未保存的知识点和主课程页面，避免服务器旧草稿导致触发点错位。进入最终课程生成前，系统检查方案是否确认，以及每个启用分支是否绑定有效主页面。

## 触发模型

每个分支拥有明确触发器：

- `afterSceneId`：学生完成哪一页后判定。
- `beforeSceneId`：分支完成后回到哪一页。
- `evidenceRule`：仅按层次、基础层或低分、优秀层且高分。
- `scoreThreshold`：节点小测阈值。
- `minimumRemainingSec`：允许插入所需的最少剩余时间。

运行时按“到达页面 → 检查个体启停 → 读取前测或教师层次 → 检查节点分数 → 检查时间预算 → 插入分支”的顺序决策。候选分支仍在运行时生成具体 PPT、互动和 TTS，不提前占用全部生成资源。

## 教师授课监控

每名学生显示三类信息：

- 学习类别：基础生、平均生、优秀生。
- 判定来源：前测自动判定或教师人工调整。
- 响应状态：等待前测、监测触发点、分支生成中、分支学习中、已响应、个体已关闭。

教师可单独关闭或重新开启自适应路径，也可把层次切换为人工值或恢复自动前测。教师人工层次在学生重新提交前测时保持优先，直到教师恢复自动模式。

## 生成与验证

最终课程生成提示包含教师确认的完整自适应模型，同时明确候选分支不得混入所有学生的固定主场景。课程预览页复用只读课程线路，展示最终主课与候选资源位置。测试覆盖精确页间触发、个体关闭、教师响应状态和备课界面完整线路；最后执行 TypeScript、相关单元测试与生产构建。
