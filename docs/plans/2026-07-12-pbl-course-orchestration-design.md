# PBL 六阶段课程编排设计

## 目标

把新课堂模式从“页面拼接一段规则，再让通用生成器猜测”改成可验证的课程配置与生成契约。教师创建项目时明确声明这是“个人项目 + AI 伴学小组”课程，配置过程证据、作品/表达/反思三段成果和伴学角色；生成器使用独立的 PBL 六阶段模板，输出带有阶段、受众和生成目的的场景；生成后用结构化检查提示缺失阶段，但不要求每个阶段都生成 PPT。

## 方案选择

采用“领域配置 + 模板规则 + 生成元数据 + 后置覆盖检查”的四层结构。课程配置属于 openPBL session 数据，负责保存教师意图；PBL 模板属于 OpenMAIC prompt 库，负责把意图翻译成严格的输出约束；场景元数据随 outline 和 scene 一起保存，负责让学生课堂与教师资源分流不再依赖标题位置；覆盖检查是纯函数，负责在流式大纲完成和最终生成完成两个节点给出可读诊断。

不把六阶段规则继续放回 `requirement` 长文本中，也不把所有阶段强行转换为学生 slide。阶段一只生成教师可讲授的项目启动资源；阶段二生成学生使用的 slide、quiz、interactive；阶段三至六生成教师主持支架和伴学引导提示，必要时使用教师资源场景承载脚本，但这些场景明确标记为教师侧，不进入学生 AI 授知课堂。

## 数据模型

`Course.pblConfig` 保存：`projectMode: "personal"`、`evidenceRequirements`、`outcome`、`companionIds`、`evaluationModel: "tri-party"` 和 `generationTemplate: "pbl-six-stage"`。过程证据采用稳定的 kind（构思草稿、方案修订记录、反思日志、数据/测试截图、AI 决策记录、作品版本），并可标记 required 和关联阶段。成果使用三个独立字段：`artifact`、`presentation`、`reflection`，避免把复合成果压缩成一个文本框。

六阶段定义与覆盖规则放在单独模块中，包含阶段 key、标签、允许的输出受众、是否需要学生学习内容以及是否需要关键教师资源。当前关键教师资源检查只要求项目启动和成果汇报有教师资源；其他阶段检查“有支架/伴学支持”，不要求生成固定 PPT。旧课程没有 `pblConfig` 时由归一化逻辑补默认值，旧的 `groups/groupId` 兼容存储继续保留为每名学生的个人项目键。

评价模型沿用现有三条 flow，但在 PBL 配置和模板中明确语义：AI 评过程，教师评成果与表达，学生评自己成长。过程证据会被注入 AI flow 与模板，学生反思不计入总分；教师仍可在课程核查页调整权重和维度。

## 生成与分流

`pbl-six-stage` 模板输出 wrapper JSON，除 `type/title/description/keyPoints` 外，必须输出 `stageKey`、`stageLabel`、`audience`、`generationPurpose`、`companionIds` 和需要时的 `resourceTypes`。模板直接内置六阶段分工、个人项目边界、三方评价和证据要求。浏览器只传课程事实与结构化 `pblProfile`，不再传一串分流规则。

场景生成阶段把当前场景的阶段、受众、生成目的、伴学角色和证据要求注入 slide/quiz/interactive/action prompts。场景构造器把这些字段写入场景元数据。分流器优先读取显式 `audience`/`stageKey`，仅对历史数据保留标题兼容逻辑。覆盖检查按显式 stageKey 汇总，并给出三类结果：缺失阶段、缺少关键教师资源、违反分流规则。检查结果不会把可用课程直接判为失败；它会在备课页和生成完成页提示教师。

## 交互与错误处理

创建页增加一个独立的“新课堂 PBL 配置”面板：顶部锁定个人项目模式；中部勾选过程证据；右侧选择 AI 伴学角色并强制保留“记记”；底部填写作品、表达、反思。评价模型以只读卡片展示，教师不需要重复配置规则。表单提交时保证必选字段和三个成果字段有值，数据以结构化对象保存。

若教师尚未生成六阶段大纲，生成按钮可继续，但会展示缺失项；若缺少阶段二学生学习场景或关键教师资源，最终发布清单继续阻止发布。LLM 返回旧格式时保留兼容解析，并按索引补阶段；新格式优先采用显式元数据。模板不存在、字段非法或场景没有学生学习内容时沿用现有错误提示路径，并把具体阶段名放入错误信息。

## 测试策略

先为配置默认值/归一化、覆盖检查、场景显式分类和 prompt 选择写单元测试；再为创建页和生成页添加最小 React 行为测试，验证个人项目声明、证据选择、成果三字段和覆盖提示。最后运行定向 Vitest、全量测试、`tsc --noEmit`、ESLint 与 Next 16 build。UI 改造完成后启动开发服务器做教师创建页、课程核查页和生成完成页的桌面/窄屏检查。
