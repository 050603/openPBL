# 自适应学习端到端设计

## 目标

把自适应学习从“运行时临时判断并生成”升级为一条可解释、可预览、可控制的完整链路：

1. 备课时教师先确认主课程，再指导补基础/拓展分支内容。
2. 生成课堂资源时同时生成教师已确认的分支成品。
3. 教师在发布前直接打开分支课堂，检查 PPT、讲稿、互动与 TTS。
4. 学生到达触发点后直接进入已准备资源，不再等待现场生成。
5. 教师授课时看到每名学生“可激活的触发点”、条件实际值和执行进度。

## 方案比较

### 方案 A：继续完全按需生成

优点是备课生成快、只生成实际使用的资源。缺点是教师无法预审，学生触发后需要等待，
且生成失败会直接打断课堂。不采用。

### 方案 B：发布前生成全部分支

教师可完整预审，学生端零等待，但任何一次资源失效都会阻断分支。可作为主路径，但需要
运行时兜底。

### 方案 C：预生成主路径 + 按需生成兜底（采用）

教师在备课大纲中填写生成指导；主课堂生成完成后，以最多两个并发任务生成全部已确认
分支。每个分支保存资源状态和课堂 ID。运行时优先复用已准备课堂；缺失或失败时才走现有
按需生成管线。该方案不引入新生成服务，复用现有 PPT、互动、TTS 和播放器。

## 统一状态模型

每个分支包含：

- `generationGuidance`：教师对案例、难度、讲解方式、禁用内容的指导。
- `preparedResource.status`：`generating / ready / failed`。
- `preparedResource.classroomId`：可直接播放和预览的课堂资源。
- `preparedResource.generatedAt / scenesCount / error`：生成审计信息。

学生运行记录仍保存在 `branchRuns`，但课堂 ID优先取预生成资源。编辑分支教学内容后清空
旧资源状态，避免教师修改大纲却继续播放旧成品。

## 触发判定

判定只使用一套纯函数，运行时和教师审计共同调用：

1. 课程与学生路径已开启。
2. 分支已由教师确认且成品资源可用（资源缺失时允许进入兜底生成）。
3. 学生已到达配置的主课页面。
4. 该分支未执行。
5. 当前层次属于目标层次。
6. 当前节点小测优先，其次最近同知识点小测，最后回退到前测分数。
7. 自适应预算剩余秒数同时满足分支时长和最小剩余时间。

审计界面只展示当前学生层次可能激活的分支。例如优秀生只看拓展分支。每个条件同时展示
“要求”和“当前实际值”，并用已满足条件数形成触发进度。时间明确标为“自适应预算”，
避免误解为整节课倒计时。

## 失败与恢复

- 某个分支预生成失败不影响主课堂生成；发布前显示失败状态和错误原因。
- 运行时资源不存在时回退到按需生成。
- 分支生成或播放失败写入学生 `branchRuns`，教师审计可定位。
- 教师关闭个体路径或调整层次后，后续触发立即使用新状态；已经完成的分支不重复播放。

## 非功能要求

- 主课堂生成不能因单个分支失败而整体失败。
- 同时最多生成两个分支，避免压垮模型/TTS服务。
- 判定函数无网络依赖、结果可复现，实际值和运行时使用值完全一致。
- 分支预览使用教师预览模式，不写学生进度。
