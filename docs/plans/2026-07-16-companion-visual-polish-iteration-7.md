# 伴学教室视觉精修第 7 轮：状态叙事与角色现场感

## 目标

继续把伴学现场做成以课堂环境和 AI 组员为主角的工作空间。本轮不新增第三方角色包，重点把真实运行时状态转换成可被看见的动作、现场提示和可操作控件。

## 本轮设计

- 角色在 `reading / ideating / questioning / planning / reviewing / noting` 六种工作习惯上保留不同的手势和桌面细节。
- `preparing / speaking / waiting / completed` 由真实运行时状态叠加到角色的姿态、呼吸、语音波形和透明度上。
- 场景底部的课堂节奏提示改为实时现场提示：调度中、回应中、当前任务和待确认动作均来自运行时或会话数据。
- SSE 进行时显示“停止回应”，直接调用既有 runtime stop，不创建第二套请求控制器。
- 侧栏、任务本和底部输入区继续作为辅助层，课堂背景与角色工作位保持主导。

## 验收

- 桌面端场景仍能同时看到角色、工作位、阶段目标和侧栏。
- 390px 窄屏无横向溢出，课堂提示、侧栏和输入区不重叠。
- 角色点名仍能切换输入目标并显示 popover。
- 关键状态不依赖动画；减少动画设置下保留静态状态文字和操作按钮。
- 目标测试、TypeScript、ESLint 与生产构建通过。
