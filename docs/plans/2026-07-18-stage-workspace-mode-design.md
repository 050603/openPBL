# 六阶段学生课堂模式控制设计

## 目标

教师可以在备课、发布前预览和授课现场为六个阶段分别指定学生只能进入普通模式、只能进入伴学模式，或允许学生自主切换并指定默认模式。授课现场修改后沿用现有会话同步链路实时下发。

## 架构

课程对象保存 `stageWorkspacePolicies`，每个阶段对应一条 `access + defaultMode` 策略。策略是界面权限的单一事实源；学生浏览器只保存该阶段在“自主切换”时的个人选择。教师强制模式时直接覆盖个人选择，恢复自主切换时仍可使用该学生原来的选择。

普通任务界面和伴学界面继续挂载在同一个 `CompanionRuntimeProvider` 下，只通过 `hidden` 控制可见性。两种界面共同读写同一个 `Course`：项目文档、画板、上传、学习事件、伴学线程、智能体任务和上下文不会因界面切换而复制或清空；尚未保存的 React 组件状态也不会因切换被卸载。

## 界面

- 备课课程大纲页：显示六阶段配置矩阵，可逐阶段配置。
- 发布前预览页：再次显示同一配置并允许修改，发布清单确认六阶段均有有效策略（缺省策略自动按“自主切换，默认伴学模式”解释）。
- 教师授课页：右侧控制面板顶部显示当前阶段模式控制，修改后调用课程更新接口；学生端通过现有 1.5 秒轮询收到更新。
- 学生课堂页：强制策略下不显示切换入口；自主策略下显示双向切换入口，并按阶段默认值初始化。

## 容错与迁移

前两个阶段（项目启动、AI 授知）固定使用普通模式，不开放伴学入口。第三至第六阶段的旧课程没有配置时使用向后兼容默认值：学生可自主切换，默认伴学模式。无效或已删除的阶段配置会被忽略。策略只控制呈现，不迁移、不删除任何学习数据。

## 验证

单元测试覆盖策略归一化、强制模式、自主默认值与阶段隔离；页面测试覆盖教师策略实时改变后界面切换、切换时共享运行时不卸载、未保存草稿仍存在；最后执行相关 Vitest、TypeScript 和 ESLint 检查。
