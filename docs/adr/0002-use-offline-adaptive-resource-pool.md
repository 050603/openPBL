# ADR-0002：自适应课堂采用完整离线资源池

## 状态

Accepted，取代 ADR-0001 中“运行时生成兜底”的部分。

## 背景

AI 课堂资源包含 PPT、讲稿、TTS 和互动内容，生成时间不可预测。即使正常情况优先使用备课成品，只要课堂仍允许现场生成，就会在资源缺失时打断学生学习。触发点还同时使用大纲 ID 和播放器场景 ID，导致资源重新生成后可能无法命中。

## 决策

- 所有启用且确认的自适应分支在备课阶段生成。
- 任一分支成品未就绪时阻止发布。
- 学生课堂只选择和播放已发布资源，不调用生成接口。
- 触发条件绑定稳定 `outlineId`；运行时 `scene.id` 仅用于播放和诊断。
- 资源可用性成为显式触发条件并进入教师审计。

## 结果

### 正面

- 课堂触发后可立即进入学习。
- 路径可复现、可预览、可审计。
- 教师对所有可能出现的内容拥有发布前控制权。
- 课堂资源重新生成不会破坏触发锚点。

### 负面

- 备课生成时间和存储量增加。
- 课堂只能组合教师已准备的资源。
- 分支生成失败会阻止发布，需要教师处理。

### 中性

- 学生之间仍可形成不同路径，但差异来自资源排列组合而非现场内容生成。

## 备选方案

- 实时生成：延迟和失败风险过高。
- 预生成加运行时兜底：仍可能破坏课堂连续性。
- 只生成通用资源、不绑定节点：教师难以判断内容何时进入主课，审计能力不足。
